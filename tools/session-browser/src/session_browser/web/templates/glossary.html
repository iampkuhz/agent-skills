Token Overview

All input-side tokens include three components: fresh (new, uncached), cache read (served from cache), and cache write (written to cache for future use). Both cache read and cache write are input-side concepts — they describe how the prompt/context is handled, not the model's output.

Different providers report these fields differently. Anthropic exposes all three; OpenAI does not report cache write; Codex only reports a total. See Provider Mapping below.

Token Components

Raw token fields from provider APIs. Input-side = fresh + cache read + cache write.

Derived Metrics

Computed from raw fields. All defined in index/metrics.py. Division by zero returns —.

Metric Formula Interpretation
Cache Reuse cache_read / (fresh + R + W) How much input is from cache. Higher = efficient.
Cache Write Ratio cache_write / (fresh + R + W) Fraction building cache. High = active cache building.
Output Ratio output / (fresh + R + W) Output-to-input. Very low (<5%) may mean context bloat.
Tools/Round tool_calls / assistant_msgs Avg tool calls per round. High may mean complex tasks.
Tokens/Round (input_side + output) / assistant_msgs Avg tokens per round. High = large context window.
Tokens/Minute (input_side + output) / (duration / 60) Token throughput. Low + long duration = possible idle.
Failed/Session failed_tools / sessions Avg failed tool calls per session. Not a percentage.

Known Limitations
# Limitation Impact
1 Token counting varies by provider Anthropic uses cl100k-based encoding; OpenAI uses its own. Cross-provider comparisons should be approximate.
2 OpenAI/Codex missing cache write These providers do not expose cache write fields. Input-side total excludes this component — not zero, just unreported.
3 Partial parse sessions Corrupted logs or format changes may cause incomplete parsing. Token counts may be underestimated.
4 Duration accuracy Derived from log timestamps. May include idle time from process suspension, laptop sleep, or rotation gaps.
5 Codex token estimation Only tokens_used (total) available. Per-round breakdown distributes evenly — an approximation.
6 Anomaly thresholds are heuristic Uses P90/P95 with 20-session minimum. Below that, fallback values apply. May not fit all workflows.
