Token Glossary

Definitions of all token metrics reported by providers. Use this to understand what each number means in the dashboard and session views.

Input Tokens

Tokens sent to the model as input (prompt). Split into fresh, cache read, and cache write categories.

Metric Definition Anthropic Field OpenAI Field
Input Fresh New tokens that were not cached. The actual "new" input cost. input_tokens input_tokens - cached_tokens
Cache Read Input tokens served from cache (read hit). Cheaper than fresh input. cache_read_input_tokens cached_tokens (prompt)
Cache Write Input tokens written to cache this turn. One-time cost for future reads. cache_creation_input_tokens cached_tokens (completion)
Total Input inputFresh + cacheRead + cacheWrite. All input-side tokens. Sum of above 3 fields Sum of above 3 fields
Output Tokens

Tokens generated by the model as output (completion). Split into visible, reasoning, and thinking categories.

Metric Definition Anthropic Field OpenAI Field
Output Visible Tokens visible in the chat output (not reasoning/thinking). output_tokens completion_tokens - reasoning_tokens
Reasoning Tokens used for internal reasoning/thinking (o1/o3 models). Not shown to user. output_tokens (not separated) reasoning_tokens
Total Output outputVisible + outputReasoning. All output-side tokens. output_tokens completion_tokens
Token Precision

How accurately the token count is reported.

Precision Meaning When Used
provider-reported Exact count from the provider's API. Most accurate. Anthropic, OpenAI, qwen-anthropic-compatible
estimated Derived from available fields. May not match provider's internal count. When only total tokens available (Codex)
unknown No token data available. Shows as — in UI. Missing or empty usage data
Common Pitfalls

Cache read ≠ output cache

cache_read_input_tokens are input-side cache hits (reading from cache). cache_creation_input_tokens are also input-side (writing to cache). Neither refers to output caching.

Don't double-count cache tokens

cache_read_input_tokens is NOT part of input_tokens. They are separate fields. Total input = input_tokens + cache_read_input_tokens + cache_creation_input_tokens.

— means unknown, not 0

When a token field is missing or the precision is unknown, the UI shows — instead of 0. This prevents misleading "zero token" impressions.

Estimated ≠ provider-reported

When only total tokens are available (e.g., Codex), we distribute them evenly across rounds. These are estimates, not exact counts from the provider.
