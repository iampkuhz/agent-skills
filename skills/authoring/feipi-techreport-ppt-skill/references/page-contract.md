# Page Contract 生成规则

## 目的

- 生成前锁定内容范围，防止 LLM 擅自扩写或缩减。
- 防止一页塞太多内容，保持 CTO 可读性。
- 让用户明确确认"这一页到底讲什么、不讲什么"。

## 固定格式

必须使用以下格式，不要自创字段：

```
## Page Contract

### 1. 本页目标
面向 CTO，用一句话说明本页要解决的认知问题。

### 2. 一句话结论
页面最终要传达的核心判断。

### 3. 使用的原始信息
列出将被使用的用户原始信息。不得添加用户未提供的信息。

### 4. 页面内容范围
本页包含：
- ...

本页不包含：
- ...

### 5. 推荐页面结构
- 主图：...
- 表格：...
- 文字：...
- 结论区：...

### 6. 版面蓝图
- 主视觉区域：...
- 证据/说明区域：...
- 脚注/来源区域：...
- 需要压缩或不进入主画面的信息：...

### 7. 生成前确认
请确认是否按这个内容范围生成当前页 PPT。
```

## 规则

- **不写风格候选**：不给出"方案 A / 方案 B"让用户选。
- **默认只给一个推荐结构**：基于内容判断，内部选择最合适版式。
- **可以说明为什么这样设计**：用一两句话解释选择理由。
- **不超过 600 中文字**：保持 Page Contract 精简。
- **信息过多时建议缩页或拆页**，但不能自动拆页，必须问用户。
- **不得添加用户未提供的信息**：第 3 节"使用的原始信息"必须逐条对应用户输入。
- **必须说明版面预算**：尤其是主视觉、证据区、脚注区如何互不冲突。
- **必须显式降维**：如果原始信息超过表格/图形容量，写清哪些信息压缩、合并或不进入主画面。
- **禁止承诺全量塞入**：除非用户明确要求附录页，否则不要承诺在一页保留全量矩阵、长脚注或长段落。

## 版面预算判断

生成 Page Contract 时先做内部判断：

| 风险 | 判断标准 | Contract 中的处理 |
|------|----------|-------------------|
| 大表风险 | 超过 4 列 × 5 行 | 主表降维，保留关键维度 |
| 长文风险 | 单个区块超过 5 条 bullet | 合并为 3-4 条判断 |
| 脚注风险 | 超过 3 条脚注 | 编号压缩或建议拆页 |
| 多中心风险 | 同时有表格、图、卡片、长脚注 | 只保留一个主视觉 |
| 溢出风险 | 需要小于字号下限才能放下 | 建议拆页 |

## 何时生成

- 信息充足性检查通过后。
- 用户未进入 fast mode。
- 用户未明确要求跳过确认。

## Page Contract 与 Slide IR 的映射

Page Contract 确认后，生成阶段必须先形成 Slide IR（中间表示），再进入 PPTX 写入。

映射关系：

| Page Contract 字段 | Slide IR 字段 | 映射规则 |
|-------------------|---------------|----------|
| 1. 本页目标 | `canvas` + `layout_pattern` | 目标决定版式选择（architecture-map、comparison-matrix 等） |
| 2. 一句话结论 | `takeaway` | 直接映射，文字可精简但不可改变含义 |
| 3. 使用的原始信息 | `source_summary` + `provenance` | 每条原始信息生成 source_item 和 provenance 条目 |
| 4. 页面内容范围 | `elements` | "包含"的条目生成元素，"不包含"的不得生成 |
| 5. 推荐页面结构 | `layout_pattern` + `regions` | 版式决定区域划分，主图/表格/文字/结论区对应 region |
| 6. 版面蓝图 | `regions[].bounds` + `elements[].layout` | 空间比例转为具体坐标 |

**核心约束**：

1. Slide IR 不得引入 Page Contract 未确认的事实。
2. 每个 element 的 `source_refs` 必须能追溯到 `source_summary` 中的原始信息。
3. 如果 Page Contract 中声明"本页不包含 X"，Slide IR 中不得出现与 X 相关的元素。
4. Slide IR 生成后需通过 `scripts/validate_slide_ir.js` 校验。

详细 Slide IR schema 见 `references/slide-ir.md` 和 `schemas/slide-ir.schema.json`。

## 容量预算（生成 IR 前必须做）

Page Contract 确认后、生成 Slide IR 前，必须先做容量预算：

1. 根据选择的 `layout_pattern`，对照容量上限（详见 `references/slide-ir.md` 的「容量预算」章节）。
2. 如果预估元素数量超过上限，必须在 Page Contract 阶段就降维——压缩、合并、建议拆页。
3. **不要**在生成 IR 后再发现过载；容量预算应在生成前完成。
4. 可使用 `helpers/ir/capacity.js` 做自动容量检查。

## 何时更新

- 用户对 Page Contract 提出调整要求时。
- 用户补充了新的原始信息时。
- 更新后重新输出完整 Page Contract，不要只输出 diff。
