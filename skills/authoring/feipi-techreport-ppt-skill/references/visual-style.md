# 视觉规范

## Style Lock（风格真源）

本 skill 使用 style lock JSON 作为长期风格真源，定义在 `templates/style-locks/` 目录。

模板和 backend 应优先消费 style lock token，不在代码中散落硬编码的颜色、字号、间距值。

可用 style lock：
- `cto-technical-report.style-lock.json` — 默认 CTO 技术汇报风格
- `architecture-blueprint.style-lock.json` — 架构蓝图风格
- `comparison-brief.style-lock.json` — 对比简报风格

加载方式：
```js
const { loadDefaultStyleLock, resolveStyleLock } = require('../helpers/style/style-lock.js');
const styleLock = resolveStyleLock(slideIR); // 从 IR backend_hints 获取
```

若 Slide IR 未指定 `backend_hints.style_lock`，默认使用 `cto-technical-report`。

校验入口：
```bash
node scripts/validate_style_lock.js templates/style-locks/cto-technical-report.style-lock.json
```

详细 schema 定义见 `schemas/style-lock.schema.json`。

## 默认风格

正式技术汇报风格。克制、清晰、结构化、高信息密度。不要营销化，不要花哨动效，不要随意插图，不要 AI 味很重的装饰。

## 视觉目标

默认按“管理层技术评审页”设计，而不是按“资料归档页”设计。

一页只能承载一个主判断：

- 标题负责提出主题。
- 一句话结论负责给出判断。
- 主视觉负责证明判断。
- 证据区负责补充 3-5 个关键事实。
- 底部或侧边负责承载 takeaway / 风险 / 脚注摘要。

任何元素如果不能服务这个主判断，都应压缩、合并、移到备注或建议拆页。

## 颜色

默认：

- **背景**：白色 / 极浅灰。
- **主色**：深蓝灰 / 深石墨。
- **辅助色**：浅蓝灰。
- **强调色**：单一强调色，用于重点、箭头、风险或结论。

不要让用户选择颜色。除非用户主动指定品牌色。

## 版面预算

16:9 单页默认按 13.33 inch × 7.5 inch 思考。

推荐空间分配：

| 区域 | 占比 | 说明 |
|------|------|------|
| Header | 12-18% | 标题 + 一句话结论，不能超过两行 |
| Main Visual | 48-62% | 主图、核心对比、流程或架构，必须是视觉中心 |
| Evidence / Side Panel | 18-28% | 关键证据、风险、取舍、解释 |
| Footer / Takeaway | 6-10% | 结论、脚注摘要、来源说明 |

硬规则：

- 主视觉和侧栏不能重叠，侧栏不能浮在表格上方。
- 脚注必须有独立区域，不得压在表格、图、卡片上。
- 如果底部需要超过 3 条脚注，必须压缩成编号说明或建议拆页。
- 页面右侧辅助卡片不能覆盖主体内容；如果主体宽度不足，改成上下结构或拆页。

## 字号

建议范围：

| 元素 | 字号 |
|------|------|
| 标题 | 26-34pt |
| 副标题/一句话结论 | 14-18pt |
| 区域标题 | 13-16pt |
| 正文 bullet | 10-13pt |
| 表格正文 | 8.5-11pt |
| 图中标签 | 8.5-11pt |
| 脚注 | 7.5-9pt |

**硬规则**：

- 正文不低于 10pt。
- 表格正文不低于 8.5pt。
- 图中标签不低于 8.5pt。
- **不要通过无限缩小字号解决信息过载。**
- 标题不能因为内容多而缩到 24pt 以下；标题过长时改写标题。
- 脚注不低于 7.5pt，且只能放次要来源、假设、编号说明。

## 密度

一页最多：

- 1 个主图。
- 1 个小表格或 3-5 个 KPI cards。
- 3-5 条 bullet。
- 1 条 takeaway。

表格建议不超过 5 行 × 4 列。

流程图建议不超过 7 个主步骤。

架构图建议不超过 9 个主节点。

## 对比页密度上限

对比页最容易退化成大表，必须先降维。

默认主表上限：

- 对比对象：最多 4 个主体列。
- 对比维度：最多 5 个主维度。
- 每个单元格：最多 2 行，每行不超过 12 个中文字符或 18 个英文字符。
- 高亮列：最多 1 列，不能整列深色导致页面压迫。

如果原始材料超过上限：

1. 先选出最能支撑结论的 4-5 个维度。
2. 把长文本改成短标签 + 数据。
3. 把次要对象合并为“其他竞品”或移入备注。
4. 用 KPI cards 展示最关键的 3-4 个差异。
5. 将全量矩阵作为“建议拆页/附录”，不要塞进当前页。

示例：6 个对象 × 10 个维度的链性能对比，不应直接生成 60 格大表。应改成：

- 顶部 3 个 KPI cards：TPS、确认时间、节点成本/配置。
- 中部 4 列 × 5 行重点矩阵：只保留支撑结论的维度。
- 右侧取舍卡：为什么目标定位成立、当前缺口在哪里。
- 底部脚注区：编号说明，不覆盖主体。

## 排版

- 主图必须是视觉中心。
- 页面必须有明确 takeaway。
- 同类元素必须网格对齐。
- 卡片间距一致。
- 不要堆满整页，保留呼吸感。
- 页面边距至少 0.3 inch，优先 0.4-0.5 inch。
- 内容区块之间至少 0.18-0.25 inch。

## 美化规则

克制不等于粗糙。默认应使用以下设计手段提升完成度：

- 用浅色页面底和白色内容区形成层次，但不要把整个页面做成一张大卡。
- 用细分隔线、色带、编号、标签、KPI cards 建立扫描路径。
- 用单一强调色标记推荐方案、目标状态或最大差异。
- 表格只保留轻量网格线，避免重边框。
- 重要数值用加粗或色块强调，解释文字降级为小字。
- 图表、表格、卡片必须共享同一基线和间距系统。

## 禁止模式

- 一整页只有大表，没有结论区和主次层级。
- 表格下方脚注与表格内容重叠。
- 右侧说明卡覆盖主表或漂浮在内容之上。
- 空白集中在页面下半部分，上半部分过度拥挤。
- 为了容纳内容而混用多套字体、字号、颜色。
- 使用大面积深色高亮导致页面左重右轻。
