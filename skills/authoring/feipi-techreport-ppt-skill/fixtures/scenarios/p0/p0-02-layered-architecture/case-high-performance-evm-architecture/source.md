这一页要说明高性能 EVM 公链的目标部署架构，以及从共识、网络、执行、RPC、硬件 OS 等层面需要改造的关键点。

### Validator 1 内部分层

1. 共识层

## 右侧改造类型说明

右侧需要是「改造类型（一级分类 -> 二级分类）」说明，按 A/B/C/D 分组，不要写成大段文字。

### A 生态演进方向

- A1 共识、执行解耦：客户端模块化设计，可插拔替换，PBS。
- A2 扩容能力支持：支持子网、Rollup 等扩容方向。

### B 共识层优化

- B1 共识算法优化：低延迟 BFT、投票聚合、阶段压缩。
- B2 网络传播优化：交易压缩、消息中继。

### C 执行层优化

- C1 客户端语言优化：Rust。
- C2 交易池和组块优化：冲突感知、预执行、交易去重。
- C3 EVM 执行优化：JIT / AOT、并行执行、热点路径优化。
- C4 存储优化：状态 DB 访问优化、缓存、异步 IO。
- C5 费用优化：定价优化、费率市场、热点限流。
- C6 交易受理改善：批量交易、专用 lane。

### D 基础设施优化

- D1 RPC 受理与处理：读写分离、索引、监控管线。
- D2 硬件与 OS 优化：CPU、内存、网络、内核优化。
