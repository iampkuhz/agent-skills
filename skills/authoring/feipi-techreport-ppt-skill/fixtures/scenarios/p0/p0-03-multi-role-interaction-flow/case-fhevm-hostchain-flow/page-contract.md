# Page Contract: P0-03 多角色交互流程 / 泳道流程页

## Layout

- 16:9 单页。
- 主流程图区占页面 70% 左右。
- 右侧说明区占页面 20%–25%。
- 顶部可放图例。

## Required Regions

1. Title region
2. Legend region
3. Multi-role flow region
4. Step explanation region
5. Key explanation region

## Flow Requirements

- 至少 5 个角色/组件。
- 步骤编号必须连续。
- 必须区分至少 3 类箭头语义。
- 箭头方向必须能表达时序。
- 右侧必须有步骤说明。
- 复杂组件可以内部再分 2–4 个子组件，但不能过密。

## Style

- 主交易线、密文计算线、解密返回线、内部同步线用不同视觉样式。
- 步骤编号使用圆形或醒目标识。
- 角色容器边界清晰。
