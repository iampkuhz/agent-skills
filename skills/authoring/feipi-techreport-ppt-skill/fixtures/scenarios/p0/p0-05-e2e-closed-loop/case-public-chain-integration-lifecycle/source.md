## 右侧接链重点技术事项

右侧上半部分需要展示 3 个重点技术事项。每个事项都要包含检查点和风险提示。

### 数据可信

- 检查点：多源交叉校验、关键数据抽检与对账、必要时自建节点。
- 风险提示：不能只信单一 RPC。

### 密钥安全

- 检查点：MPC / HSM、冷热钱包分层、权限审批与签名策略。
- 风险提示：资产系统最高风险点。

### 资产 / 应用操作安全

- 检查点：合约校验与白名单、异常转账拦截、余额 / 事件回放与审计。
- 风险提示：避免误操作与合约风险。

## 右下高 TPS 链 3 个重点投入项

右下需要单独突出高 TPS 链接入时的 3 个重点投入项。每个事项同样要包含检查点和风险提示。

### 链访问容量

- 检查点：多 RPC / 限流降级、缓存批处理、必要时自建节点。
- 风险提示：RPC 限流、延迟、成本高、单点依赖。

### 高吞吐 Indexer

- 检查点：实时订阅、断点续扫、历史回放。
- 风险提示：事件延迟、漏记账、余额不一致。

### 交易落地率

- 检查点：交易队列、费用估算、重试补偿。
- 风险提示：已提交但资产系统无法确认完成。
