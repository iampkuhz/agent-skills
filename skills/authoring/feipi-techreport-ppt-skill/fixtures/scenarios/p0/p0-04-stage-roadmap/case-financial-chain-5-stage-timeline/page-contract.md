# Page Contract: P0-04 阶段演进 Roadmap 页

## Layout

- 16:9 单页。
- 横向 5 个阶段卡片等宽排列。
- 顶部标题 + 结论条。
- 中部为阶段定位 / 核心指标。
- 下方分两到三层展示技术演进、金融增强、场景支持。

## Required Regions

1. Title region
2. Conclusion region
3. Stage KPI row
4. Performance evolution row
5. Financial enhancement row
6. Scenario / ecosystem support row
7. Footnote region

## Roadmap Requirements

- 必须有 5 个阶段。
- 每个阶段必须包含时间窗口和核心指标。
- 每个阶段必须包含性能升级与金融增强。
- 阶段卡片必须横向对齐。
- 关键 TPS 数字需要显著。
- 每阶段文本结构应保持一致。

## Style

- 不同信息层可以使用浅色区域区分。
- 阶段序号使用圆形编号。
- 关键 TPS 使用大字号。
- 说明文字短句化。
