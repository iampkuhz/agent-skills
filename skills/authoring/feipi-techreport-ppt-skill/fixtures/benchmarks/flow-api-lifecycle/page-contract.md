## Page Contract

### 1. 本页目标
展示 API 网关从请求到响应的完整生命周期，突出关键决策点。

### 2. 一句话结论
认证和限流是请求路径上的两个关键卡点，决定请求是否进入业务逻辑。

### 3. 使用的原始信息
- API 网关流程描述（6 个阶段）
- 每个阶段的输入输出
- 关键决策点标注

### 4. 页面内容范围
本页包含：
- 6 步流程图，从左到右
- 每个阶段的简要说明
- 决策点高亮

本页不包含：
- 每个阶段的代码实现细节
- 异常处理的全部分支

### 5. 推荐页面结构
- 主图：横向 6 步流程图
- 侧栏：关键决策点说明
- 底部：结论

### 6. 版面蓝图
- 主视觉区域：6 个等距步骤节点 + 5 条连接线
- 证据区域：右侧说明每个步骤耗时和关键指标
- 需要压缩的信息：异常分支统一归入脚注
