## Page Contract

### 1. 本页目标
用决策树展示单体到微服务迁移的关键判断点和各路径利弊。

### 2. 一句话结论
当前团队规模和业务增速下，渐进式拆分（先拆最高频服务）是风险最低的路径。

### 3. 使用的原始信息
- 决策因素：团队规模、业务增速、运维能力、技术债
- 三条路径：不拆分、渐进拆分、全量拆分
- 每条路径的利弊

### 4. 页面内容范围
本页包含：
- 决策树图（根节点 + 判断分支 + 叶子结论）
- 每条路径的一行利弊
- 推荐路径高亮

本页不包含：
- 每种方案的详细实施计划
- 成本估算

### 5. 推荐页面结构
- 主图：决策树，从左到右分支
- 侧栏：三条路径利弊摘要
- 底部：推荐结论

### 6. 版面蓝图
- 主视觉区域：决策树节点 + 分支线
- 证据区域：右侧路径摘要
- 需要压缩的信息：实施计划放入备注
