# {{TITLE}}

## 核心目标

用最小上下文成本完成高质量交付，并保留可验证证据。

## 适用场景

- 明确列出触发条件、输入类型和预期输出。
- 对非适用场景给出边界，避免误触发。

## 工作流（Explore -> Plan -> Implement -> Verify）

1. Explore：仅读取必要文件，避免无边界探索。
2. Plan：先定义改动范围与验收标准，再实现。
3. Implement：优先复用 `scripts/`、`references/`、`assets/`。
4. Verify：执行当前任务产物校验并记录结果。

## 标准命令（使用态）

- 只放完成用户任务所需的命令（如提取、生成、单次校验）。

## 验收标准

1. 至少有一种可执行校验方式（脚本、命令输出、截图比对）。
2. 输出必须包含“验证步骤 + 验证结果 + 剩余风险”。
3. 若验证失败，先修复再交付。

## 渐进式披露导航

- 高层说明：留在本文件。
- 细节文档：放入 `references/`，并在此处直接链接（一级引用）。
- 可执行逻辑：放入 `scripts/`，优先“执行脚本”而非现场重写。

## 资源说明

- `scripts/`: 可选，放可复用、确定性强的辅助脚本。
- `references/`: 可选，放按需读取的领域资料。
- `assets/`: 可选，放输出时要用到的模板或静态文件。
